# Как использовать пример расчета многоуровневой системы

## Файлы с примером

1. **`EXAMPLE_MULTILEVEL_LAPTOP.md`** - Подробное описание расчета с формулами
2. **`example_laptop_data.json`** - JSON-данные для тестирования в системе
3. **`example_laptop_structure.txt`** - Визуальная диаграмма структуры

## Как воспроизвести пример в системе

### Вариант 1: Ручной ввод через интерфейс

1. **Откройте приложение** и начните новый анализ

2. **Введите цель:**
   ```
   Выбор ноутбука для работы
   ```

3. **Включите многоуровневую иерархию:**
   - Поставьте галочку "Использовать многоуровневую иерархию (4+ уровней)"

4. **Создайте уровни:**
   
   **Уровень 1:** "Основные критерии"
   - Производительность
   - Стоимость
   
   **Уровень 2:** "Подкритерии"
   - Процессор
   - Оперативная память
   - Цена покупки
   - Стоимость обслуживания
   
   **Уровень 3:** "Альтернативы"
   - MacBook Pro
   - Dell XPS
   - Lenovo ThinkPad

5. **Заполните матрицы сравнения:**

   **Шаг 1:** Сравнение критериев уровня 1
   - Производительность vs Стоимость: **3** (Производительность умеренно важнее)

   **Шаг 2:** Сравнение подкритериев для "Производительность"
   - Процессор vs ОЗУ: **2** (Процессор немного важнее)

   **Шаг 3:** Сравнение подкритериев для "Стоимость"
   - Цена покупки vs Обслуживание: **5** (Цена покупки существенно важнее)

   **Шаг 4:** Сравнение альтернатив по каждому подкритерию:
   
   *По "Процессор":*
   - MacBook Pro vs Dell XPS: **1/3** (0.333)
   - MacBook Pro vs ThinkPad: **1/5** (0.2)
   - Dell XPS vs ThinkPad: **1/2** (0.5)
   
   *По "ОЗУ":*
   - MacBook Pro vs Dell XPS: **2**
   - MacBook Pro vs ThinkPad: **1/2** (0.5)
   - Dell XPS vs ThinkPad: **1/3** (0.333)
   
   *По "Цена покупки":*
   - MacBook Pro vs Dell XPS: **1/5** (0.2)
   - MacBook Pro vs ThinkPad: **1/7** (0.143)
   - Dell XPS vs ThinkPad: **1/3** (0.333)
   
   *По "Стоимость обслуживания":*
   - MacBook Pro vs Dell XPS: **3**
   - MacBook Pro vs ThinkPad: **5**
   - Dell XPS vs ThinkPad: **2**

6. **Просмотрите результаты:**
   - Система автоматически рассчитает глобальные приоритеты
   - Ожидаемый результат: ThinkPad (56.62%) > Dell XPS (26.30%) > MacBook Pro (17.07%)

### Вариант 2: Использование JSON-данных (для разработчиков)

Если вы хотите протестировать API напрямую, используйте данные из `example_laptop_data.json`:

```bash
curl -X POST http://localhost:3001/api/calculate-global-priorities \
  -H "Content-Type: application/json" \
  -d @example_laptop_data.json
```

## Проверка результатов

После расчета проверьте:

1. **Приоритеты уровня 1:**
   - Производительность ≈ 0.750 (75%)
   - Стоимость ≈ 0.250 (25%)

2. **Приоритеты уровня 2:**
   - Процессор ≈ 0.667 (66.7%)
   - ОЗУ ≈ 0.333 (33.3%)
   - Цена покупки ≈ 0.833 (83.3%)
   - Обслуживание ≈ 0.167 (16.7%)

3. **Глобальные приоритеты альтернатив:**
   - ThinkPad ≈ 0.566 (56.6%)
   - Dell XPS ≈ 0.263 (26.3%)
   - MacBook Pro ≈ 0.171 (17.1%)

## Интерпретация результатов

### Почему ThinkPad выиграл?

1. **Лучший процессор** (58.2% приоритета среди альтернатив)
2. **Лучшая ОЗУ** (54.0% приоритета)
3. **Самая низкая цена** (64.9% приоритета)
4. **Высокий вес критерия "Производительность"** (75% от общей важности)
5. **Высокий вес "Цены покупки"** (83.3% от критерия "Стоимость")

### Почему MacBook Pro проиграл?

1. **Очень высокая цена** (только 7.2% приоритета по цене)
2. **Средний процессор** (10.9% приоритета)
3. Хотя обслуживание дешевле (64.8%), но этот фактор имеет низкий вес (16.7%)

## Модификация примера

Вы можете изменить пример, чтобы увидеть, как меняются результаты:

- **Увеличьте важность стоимости:** Измените сравнение "Производительность vs Стоимость" с 3 на 1/3
- **Измените приоритеты подкритериев:** Например, сделайте ОЗУ важнее процессора
- **Добавьте больше уровней:** Например, разбейте "Процессор" на "Частота" и "Количество ядер"
